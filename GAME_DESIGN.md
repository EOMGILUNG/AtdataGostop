# 웹 맞고 게임 분석/설계

## 1. 목표

현재 폴더의 화투 이미지 자산을 사용해 브라우저에서 동작하는 2인용 맞고 게임을 만든다.

초기 목표는 “사람 1명 vs 컴퓨터 1명”의 로컬 싱글 플레이이며, 이후 온라인 대전이나 모바일 최적화로 확장할 수 있게 게임 규칙/상태/화면을 분리한다.

## 2. 현재 자산 상태

- 카드 앞면: `cards/01.png` ~ `cards/48.png`
- 카드 뒷면: `cards/back.png`
- 보너스 카드 이미지: 현재 없음. 1차 구현에서는 CSS로 그린 카드 또는 임시 이미지로 표현한다.
- 앞면 카드 크기: 대체로 250 x 407 또는 250 x 408
- 뒷면 카드 크기: 1024 x 1536
- 파일 번호는 4장 단위로 1월부터 12월까지 정렬된 것으로 확인됨

## 3. 카드 데이터 모델

게임 로직에서는 이미지 파일명을 직접 판단하지 않고, 카드 메타데이터를 별도로 둔다.

```ts
type CardType = "gwang" | "tti" | "yeol" | "pi" | "ssangpi" | "bonus";

type Card = {
  id: string;          // "01" ~ "48", "B01" ~ "B03"
  month: number | null;// 1 ~ 12, 보너스 카드는 null
  image?: string;      // "cards/01.png", 보너스 카드는 없을 수 있음
  types: CardType[];   // 예: ["gwang"], ["pi"], ["ssangpi"]
  name: string;        // 사람이 읽기 쉬운 이름
  piValue?: number;    // 피/쌍피/보너스피 계산값
  tags?: string[];     // hongdan, cheongdan, godori 등 족보 판정용
};
```

### 4.2 쌍피와 9월 술잔 처리

1차 구현 기본값:

- 9월 술잔 `35`: 열끗이면서 쌍피로 계산한다.
- 11월 오동 쌍피 `43`: 쌍피로 계산한다.
- 12월 비 쌍피 `46`: 쌍피로 계산한다.
- 쌍피는 피 점수 계산 시 2장으로 센다.
- 열끗 + 쌍피처럼 두 종류를 가진 카드는 각 점수 영역에 모두 반영한다.

옵션화 후보:

- `35`를 열끗으로만 계산
- `35`를 쌍피로만 계산
- `43` 또는 `46`의 쌍피 적용 여부 변경

### 4.3 보너스 카드 추가

현재 이미지 자산에는 보너스 카드가 없다. 1차 구현에서는 보너스 카드를 데이터로 추가하고, 화면에서는 CSS 기반 카드로 표현한다. 이후 전용 이미지가 생기면 `image`만 연결한다.

보너스 카드는 지역과 플랫폼마다 1~3장을 쓰는 경우가 있고, 일반적으로는 2장 또는 3장 구성이 많이 쓰인다. 온라인 맞고의 속도감과 피 점수 흐름을 살리기 위해 1차 기본값은 3장으로 확정한다.

| ID | 이름 | 종류 | 피 계산값 | 태그 |
|---|---|---|---:|---|
| B01 | 보너스 2피 | 보너스 | 2 | bonus-pi |
| B02 | 보너스 2피 | 보너스 | 2 | bonus-pi |
| B03 | 보너스 3피 | 보너스 | 3 | bonus-pi |

보너스 카드 기본 규칙:

- 보너스 카드는 월이 없으므로 바닥패와 월 매칭하지 않는다.
- 획득 영역에서는 피 점수에만 반영한다.
- 보너스 카드를 획득하면 상대 피 1장을 가져온다.
- 상대에게 가져올 피가 없으면 피 이동 없이 보너스 카드만 획득한다.
- 보너스 카드는 광박/피박/멍박 중 피박 판정의 피 장수에는 포함한다.
- 보너스 카드는 총통, 흔들기, 폭탄, 쪽, 따닥, 뻑의 월 판정에는 포함하지 않는다.

보너스 카드 처리 흐름:

1. 시작 바닥패에 보너스 카드가 있으면 선 플레이어가 즉시 획득한다.
2. 바닥에서 제거된 보너스 카드 수만큼 더미에서 새 카드를 바닥에 보충한다.
3. 손패에 보너스 카드가 있으면 플레이어는 자기 턴에 낼 수 있다.
4. 손패에서 보너스 카드를 내면 즉시 획득하고, 더미에서 1장을 손패로 보충한 뒤 더미 뒤집기를 진행한다.
5. 더미에서 뒤집은 카드가 보너스 카드이면 즉시 획득하고, 더미에서 다시 1장을 뒤집는다.
6. 보너스 카드 처리가 연속으로 발생할 수 있으므로 더미가 빌 때까지 반복 처리한다.

## 5. 맞고 기본 진행 규칙

### 5.1 인원

- 플레이어 1명
- 컴퓨터 1명

### 5.2 카드 분배

일반적인 맞고 기준:

- 각 플레이어 손패 10장
- 바닥패 8장
- 더미패 23장

보너스 카드 포함 기준:

- 기본 화투 48장 + 보너스 카드 3장 = 총 51장
- 플레이어 10장, 컴퓨터 10장, 바닥 8장을 나누고 남은 23장을 더미로 둔다.
- 시작 바닥패에 보너스 카드가 있으면 선이 가져가고 더미에서 바닥을 보충한다.

분배 과정:

1. 기본 화투 48장과 보너스 카드 3장을 합쳐 섞는다.
2. 플레이어/컴퓨터/바닥에 규칙 수량만큼 배분한다.
3. 남은 카드는 더미로 둔다.
4. 시작 전 바닥에 같은 월 4장이 깔리는 등 특수 상황은 룰 옵션으로 처리한다.
5. 시작 바닥패의 보너스 카드를 처리하고 바닥패를 8장으로 보충한다.

### 5.3 턴 진행

한 턴은 다음 순서로 진행한다.

1. 현재 플레이어가 손패 1장을 선택해 바닥에 낸다.
2. 낸 카드가 보너스 카드이면 즉시 획득하고 손패를 1장 보충한다.
3. 일반 카드라면 같은 월의 바닥패가 있는지 매칭 후보를 만든다.
4. 더미에서 1장을 뒤집어 바닥에 낸다.
5. 뒤집은 카드가 보너스 카드이면 즉시 획득하고 다시 뒤집는다.
6. 뒤집은 일반 카드와 같은 월의 바닥패가 있으면 매칭 후보를 만든다.
7. 획득 가능한 패를 현재 플레이어의 획득 영역으로 이동한다.
8. 점수를 다시 계산한다.
9. 7점 이상이면 `고` 또는 `스톱`을 선택할 수 있다.
10. 스톱하지 않으면 턴을 상대에게 넘긴다.

### 5.4 같은 월 매칭 규칙

- 같은 월 바닥패가 0장: 낸 카드를 바닥에 놓는다.
- 같은 월 바닥패가 1장: 낸 카드와 바닥패 1장을 획득한다.
- 같은 월 바닥패가 2장: 플레이어가 둘 중 하나를 선택해 획득한다.
- 같은 월 바닥패가 3장: 낸 카드와 바닥패 3장을 모두 획득한다.

더미에서 뒤집은 카드도 같은 방식으로 처리한다.

### 5.5 흔들기/폭탄

1차 구현에서는 흔들기와 폭탄을 모두 포함한다. 흔들기는 손패에 같은 월 3장이 있을 때 선언할 수 있는 배수 액션이고, 폭탄은 같은 월 3장을 한 번에 제출하는 턴 액션이다.

- 같은 월 3장을 손에 들고 있으면 자신의 턴 시작 시 흔들기를 선언할 수 있다.
- 흔들기를 선언하면 해당 월을 공개하고 최종 점수 배수 x2를 기록한다.
- 한 판에서 여러 번 흔들면 흔든 횟수만큼 배수를 누적한다.
- 같은 월 3장을 손에 들고 있고 그 월의 바닥패가 있을 때 폭탄을 낼 수 있다.
- 폭탄을 내면 손에 있는 같은 월 3장을 한 번에 제출한다.
- 제출한 3장과 바닥의 같은 월 패를 모두 획득한다.
- 폭탄 후 손패 부족을 보정하기 위해 더미에서 보너스 패를 가져오는 처리는 룰 옵션으로 둔다.
- 1차 기본값은 폭탄 후 즉시 더미에서 2장을 보충해 손패 템포를 맞춘다.

## 6. 점수 계산 규칙

초기 구현은 널리 쓰이는 맞고 점수 체계를 기본값으로 사용한다. 지역 룰 차이가 있으므로 모든 배수/특수 규칙은 설정값으로 분리한다.

### 6.1 광

- 3광: 3점
- 비광 포함 3광: 2점
- 4광: 4점
- 5광: 15점

비광 포함 3광은 1차 구현에서 `2점`으로 확정한다.

### 6.2 열끗

- 열끗 5장: 1점
- 이후 1장 추가마다 +1점
- 고도리: 5점

### 6.3 띠

- 띠 5장: 1점
- 이후 1장 추가마다 +1점
- 홍단: 3점
- 청단: 3점
- 초단: 3점

### 6.4 피

- 피 10장: 1점
- 이후 1장 추가마다 +1점
- 쌍피는 피 2장으로 계산
- 보너스 2피는 피 2장으로 계산
- 보너스 3피는 피 3장으로 계산

### 6.5 배수/벌칙 규칙

1차 구현에서는 기본 맞고의 긴장감을 살리기 위해 일부 박 규칙을 적용한다. 단, 계산은 `RuleOptions`로 분리해 나중에 쉽게 켜고 끌 수 있게 한다.

- 고 배수
- 광박
- 피박
- 멍박
- 흔들기 배수
- 폭탄 배수
- 나가리
- 총통

### 6.6 1차 점수/배수 규칙 확정

#### 고 배수

- 1고: 기본 점수에 +1점
- 2고: 기본 점수에 +2점
- 3고 이상: 최종 점수 배수 적용
- 3고: x2
- 4고: x4
- 이후 고 1회 추가마다 배수 x2

구현에서는 `goCount`를 따로 저장하고, 점수 계산 마지막 단계에서 반영한다.

#### 광박

적용한다.

조건:

- 승자가 광 점수로 점수를 냈다.
- 패자가 획득한 광이 0장이다.

효과:

- 최종 점수 x2

#### 피박

적용한다.

조건:

- 승자가 피 점수로 점수를 냈다.
- 패자의 피 계산 장수가 5장 이하이다.

효과:

- 최종 점수 x2

#### 멍박

적용한다.

조건:

- 승자가 열끗 점수로 점수를 냈다.
- 승자의 열끗 계산 장수가 7장 이상이다.
- 패자의 열끗 계산 장수가 0장이다.

효과:

- 최종 점수 x2

#### 흔들기/폭탄 배수

흔들기와 폭탄은 1차 구현에 포함한다.

흔들기 조건:

- 현재 플레이어 손패에 같은 월 일반 카드가 3장 있다.
- 해당 월로 아직 흔들기를 선언하지 않았다.
- 플레이어가 흔들기를 선택한다.

흔들기 처리:

- 해당 월 카드 3장을 잠깐 공개한다.
- 플레이어 상태에 `shakeCount += 1`을 기록한다.
- 흔든 월을 `shakenMonths`에 저장해 중복 선언을 막는다.
- 최종 점수 계산 시 흔들기 1회마다 x2를 적용한다.
- 흔들기 선언 후에도 반드시 바로 폭탄을 내야 하는 것은 아니다.

폭탄 조건:

- 현재 플레이어 손패에 같은 월 카드가 3장 있다.
- 같은 월의 바닥패가 1장 이상 있다.
- 플레이어가 폭탄 제출을 선택한다.

폭탄 처리:

- 같은 월 손패 3장을 모두 제출한다.
- 바닥의 같은 월 패와 함께 획득한다.
- 이후 더미에서 2장을 보충한다.
- 보충 카드가 바닥 매칭을 발생시키지 않도록 “손패 보충”으로만 처리한다.
- 폭탄 자체 배수는 적용하지 않는다.

#### 나가리

적용한다.

조건:

- 양쪽 모두 스톱하지 못한 상태로 손패와 더미가 모두 소진된다.
- 또는 점수는 났지만 스톱 없이 진행하다가 종료되고 최종 승자가 정해지지 않는다.

효과:

- 해당 판은 승패 없이 종료한다.
- 다음 판 시작 시 `nagariMultiplier`를 x2로 적용한다.
- 연속 나가리는 배수를 누적한다. 예: 2연속 나가리 x4
- 1차에서는 라운드 상태에 배수만 저장하고, 별도 재화/점수판 누적은 구현하지 않는다.

#### 총통

적용한다.

조건:

- 최초 분배 직후 한 플레이어 손패에 같은 월 4장이 모두 있다.

효과:

- 해당 플레이어는 즉시 승리하거나 계속 진행을 선택할 수 있다.
- 1차 기본값은 즉시 승리 선택만 제공한다.
- 총통 점수는 10점으로 계산한다.
- 총통 승리에도 광박/피박/멍박은 적용하지 않는다.

### 6.7 쪽/따닥/뻑/싹쓸이 구현 기준

1차 구현에서는 아래 특수 획득 규칙을 모두 적용한다.

| 규칙 | 1차 구현 | 처리 기준 |
|---|---|---|
| 쪽 | 적용 | 낸 카드가 바닥에 붙지 않고, 더미에서 같은 월이 나와 두 장을 획득하면 상대 피 1장을 가져온다 |
| 따닥 | 적용 | 같은 월 바닥패 2장에 손패 1장을 내고, 더미에서도 같은 월이 나와 4장을 모두 획득하면 상대 피 1장을 가져온다 |
| 뻑 | 적용 | 같은 월 3장 묶음이 바닥에 남는 상황을 `ppeok`으로 표시하고, 이후 그 묶음을 가져간 플레이어가 상대 피 1장을 가져온다 |
| 싹쓸이 | 적용 | 턴 처리 결과 바닥패가 0장이 되면 상대 피 1장을 가져온다 |

상대 피 1장 가져오기 기준:

- 상대 획득패 중 일반 피를 우선 가져온다.
- 일반 피가 없으면 쌍피를 가져온다.
- 상대가 가져올 피가 없으면 보너스 없이 이벤트만 기록한다.

특수 이벤트는 `GameEvent`에 명시한다.

예시:

- `special: "jjok"`
- `special: "ttadak"`
- `special: "ppeok"`
- `special: "sweep"`

## 8. 화면 구성

첫 버전은 한 화면에서 전체 게임이 보이도록 만든다. 맞고 UI의 핵심은 장식보다 판세 파악이다. 사용자가 매 턴 확인해야 하는 정보는 손패, 바닥패, 획득패, 점수, 현재 선택 가능 액션이다.

### 8.1 디자인 방향

- 분위기: 실제 화투판처럼 집중감 있는 녹색 테이블 기반
- 정보 밀도: 카드와 점수는 크게, 보조 로그와 설정은 작게
- 조작 방식: 플레이어는 손패를 클릭/탭해서 낸다
- 피드백: 선택 가능한 카드는 강조하고, 불가능한 카드는 약하게 표시한다
- 애니메이션: 카드 이동/뒤집기 정도만 짧게 사용한다

색상 초안:

- 배경: 어두운 녹색 테이블
- 카드 영역: 녹색보다 살짝 밝거나 어두운 구역 분리
- 강조색: 빨강 또는 금색
- 보조 텍스트: 연한 회색
- 위험/결정 버튼: 빨강
- 일반 버튼: 짙은 회색 또는 녹색

### 8.2 데스크톱 레이아웃 초안

### 8.3 주요 UI 영역

#### 상대 영역

- 상대 손패는 `back.png`로 표시한다.
- 카드 수를 숫자로 함께 표시한다.
- 상대 획득패는 종류별 요약으로 보여준다.
  - 광: 아이콘/작은 카드 묶음
  - 열끗: 개수와 주요 족보
  - 띠: 홍단/청단/초단 진행 상태
  - 피: 계산된 피 장수
- 상대가 방금 낸 카드는 짧게 강조한다.

#### 중앙 바닥 영역

- 바닥패는 월별로 묶어 배치한다.
- 같은 월 카드가 여러 장이면 살짝 겹쳐서 보여준다.
- 플레이어가 손패를 냈을 때 매칭 후보가 2개 이상이면 선택 가능한 바닥패에 테두리를 준다.
- 더미는 중앙 한쪽에 두고 남은 카드 수를 표시한다.
- 더미에서 뒤집힌 카드는 잠깐 별도 위치에 보여준 뒤 바닥/획득 영역으로 이동한다.

#### 플레이어 영역

- 손패는 항상 화면 하단에 고정한다.
- 클릭 가능한 손패는 살짝 위로 올라가거나 밝은 테두리를 가진다.
- 현재 낼 수 없는 상태에서는 손패 전체를 비활성화한다.
- 손패가 10장에서 줄어들어도 카드 간격이 급격히 바뀌지 않게 고정 슬롯 또는 일정 간격을 사용한다.

#### 점수/상태판

항상 보이는 항목:

- 현재 턴
- 내 점수
- 상대 점수
- 내 고 횟수
- 상대 고 횟수
- 더미 남은 장수
- 현재 라운드 상태

점수 상세를 펼치면 보이는 항목:

- 광 점수
- 열끗 점수
- 띠 점수
- 피 점수
- 고/박/배수 후보

#### 액션 로그

최근 이벤트를 짧게 보여준다.

예시:

- `플레이어: 3월 홍단 제출`
- `더미: 8월 광 공개`
- `플레이어: 광 1장 획득`
- `7점 달성: 고/스톱 선택`

로그는 게임 이해를 돕는 용도이므로 3~5줄만 유지한다.

### 8.4 모달/선택 UI

#### 같은 월 2장 선택

상황:

- 손패 또는 더미 카드와 같은 월의 바닥패가 2장 있을 때

UI:

- 화면 중앙에 작은 선택 패널을 띄운다.
- 선택 가능한 바닥패 2장을 크게 보여준다.
- 사용자가 카드 1장을 선택하면 즉시 처리한다.

#### 고/스톱 선택

상황:

- 현재 플레이어가 7점 이상 달성했을 때

UI:

- 현재 점수와 예상 배수를 보여준다.
- `고`와 `스톱` 버튼을 크게 배치한다.
- 스톱 시 예상 최종 점수를 보여준다.

#### 흔들기 선택

상황:

- 플레이어 턴 시작 시 손패에 같은 월 일반 카드 3장이 있다.
- 해당 월로 아직 흔들기를 선언하지 않았다.

UI:

- 하단 손패 바로 위에 작은 액션 바를 표시한다.
- 액션 바 문구: `같은 월 3장: 흔들기 가능`
- 버튼은 `흔들기`와 `그냥 진행` 두 개만 둔다.
- 흔들기 대상 카드 3장은 금색 테두리와 살짝 위로 올라간 상태로 강조한다.
- `흔들기`를 누르면 카드 3장이 1초 정도 공개 강조되고, 상태판에 `흔들기 x2` 배지가 생긴다.
- `그냥 진행`을 누르면 이번 턴에는 흔들기 안내를 닫고 일반 손패 선택으로 돌아간다.
- 같은 턴에 흔들기 가능한 월이 여러 개라면 액션 바에 월별 버튼을 나열하지 않고, `흔들기` 클릭 후 작은 모달에서 대상 묶음을 선택한다.

컴퓨터 처리:

- 컴퓨터가 흔들기를 선언하면 전체 모달을 띄우지 않는다.
- 상대 영역에 `상대 흔들기` 배지를 1초 정도 표시하고 액션 로그에 기록한다.
- 컴퓨터가 공개한 3장 이미지는 상대 손패 영역 위에 짧게 펼쳐 보여준 뒤 다시 뒷면으로 접는다.

#### 폭탄 선택

상황:

- 플레이어 손패에 같은 월 일반 카드 3장이 있다.
- 바닥에 같은 월 카드가 1장 이상 있다.

UI:

- 폭탄 가능한 카드 3장 묶음 위에 작은 `폭탄` 버튼을 붙인다.
- 바닥의 대상 월 카드도 빨간 테두리로 강조한다.
- 사용자가 카드 3장 중 하나를 클릭하면 일반 제출을 기본으로 하지 않고, 먼저 선택 팝오버를 보여준다.
- 팝오버 버튼:
  - `폭탄`
  - `1장만 내기`
- `폭탄`을 선택하면 3장이 한 번에 중앙으로 이동하고, 획득 카드 묶음으로 들어간다.
- 폭탄 후 더미에서 보충되는 2장은 손패 끝에 뒤집힌 상태로 들어왔다가 앞면으로 열린다.
- 폭탄 자체 배수는 없으므로 상태판에는 배수 배지를 추가하지 않는다.

컴퓨터 처리:

- 컴퓨터가 폭탄을 쓰면 중앙에 `상대 폭탄` 토스트를 띄운다.
- 폭탄 카드 3장과 바닥 대상 카드를 1초 정도 보여준 뒤 상대 획득패로 이동한다.
- 손패 보충 후 상대 손패 뒷면 카드 수가 갱신된다.

#### 총통 선택

상황:

- 최초 분배 직후 한 플레이어 손패에 같은 월 4장이 모두 있다.

UI:

- 게임 시작 처리 중 총통을 먼저 검사한다.
- 총통이 발견되면 일반 화면으로 진입하기 전에 전체화면 모달을 띄운다.
- 플레이어 총통 모달:
  - 제목: `총통`
  - 같은 월 카드 4장을 크게 표시
  - 설명: `10점 즉시 승리`
  - 버튼: `승리 확정`
- 컴퓨터 총통 모달:
  - 제목: `상대 총통`
  - 상대의 같은 월 카드 4장을 공개
  - 설명: `상대가 10점으로 승리`
  - 버튼: `다시 시작`
- 1차 구현에서는 총통 후 계속 진행 선택을 제공하지 않는다.
- 총통 승리에는 광박/피박/멍박/흔들기 배수를 적용하지 않는다.

#### 라운드 종료

표시 항목:

- 승자
- 최종 점수
- 주요 족보
- 고 횟수
- 적용된 배수
- `다시 시작` 버튼

### 8.5 모바일 레이아웃

모바일에서는 모든 정보를 한 번에 다 보여주기보다 플레이 우선순위를 둔다.

```text
┌────────────────────┐
│ 상대 점수 / 손패수 │
├────────────────────┤
│ 바닥패 / 더미      │
│                    │
├────────────────────┤
│ 내 점수 요약       │
├────────────────────┤
│ 내 손패 가로스크롤 │
└────────────────────┘
```

모바일 기준:

- 플레이어 손패는 하단 고정 가로 스크롤

- 액션 로그는 기본 접힘
- 고/스톱, 카드 선택은 전체 너비 하단 시트로 표시
- 카드 크기는 화면 너비 기준이 아니라 고정 비율로 단계별 조정한다

## 9. 컴퓨터 AI 설계

초기 AI는 규칙 기반으로 구현한다.

### 9.1 1차 AI 난이도

1차 구현의 기본 난이도는 `보통`으로 확정한다.

목표:

- 완전 랜덤보다 납득 가능한 선택을 한다.
- 사람을 지나치게 압박하지 않는다.
- 디버깅하기 쉽도록 판단 근거를 단순하게 유지한다.

우선순위:

1. 즉시 점수를 만들 수 있는 카드
2. 광/고도리/단 조합을 완성하거나 방해하는 카드
3. 피 획득량이 큰 카드
4. 상대가 가져가기 쉬운 바닥패를 줄이는 카드
5. 무작위 후보

### 9.2 난이도 옵션 설계

```ts
type CpuDifficulty = "easy" | "normal" | "hard";
```

쉬움:

- 매칭 가능한 카드 중 무작위 선택
- 매칭 카드가 없으면 완전 무작위 선택

보통:

- 점수 획득 가능성을 기준으로 카드 후보에 가중치를 준다.
- 광, 고도리, 단 완성 후보를 우선한다.
- 피 획득량이 큰 선택을 우선한다.

어려움:

- 상대 획득패를 보고 광박/피박 가능성을 방어한다.
- 상대가 완성 직전인 단/고도리를 견제한다.
- 더미는 알 수 없다는 전제에서 현재 공개 정보만 사용한다.

1차 구현은 `normal`만 실제로 사용하고, `easy`와 `hard`는 구조만 열어둔다.

향후 개선:

- 쉬움/어려움 난이도 실제 활성화
- 상대 획득패 기반 방어 전략 고도화
- 몬테카를로 시뮬레이션

## 12. 테스트 계획

규칙 로직은 UI와 별도로 테스트한다.

- 카드 수: 보너스 포함 시 항상 51장인지
- 분배 후 손패/바닥/더미 수량이 맞는지
- 시작 바닥 보너스 카드가 선에게 이동하고 바닥이 보충되는지
- 손패 보너스 카드를 내면 획득 후 손패가 보충되는지
- 더미에서 보너스 카드가 나오면 획득 후 다시 뒤집는지
- 보너스 카드가 피 점수와 피박 판정에 반영되는지
- 보너스 카드 획득 시 상대 피 1장 이동이 처리되는지
- 같은 월 매칭이 0/1/2/3장 상황에서 맞게 동작하는지
- 광/띠/열끗/피 점수가 올바른지
- 9월 술잔이 열끗 + 쌍피로 계산되는지
- 쌍피가 피 2장으로 계산되는지
- 비광 포함 3광이 2점으로 계산되는지
- 광박/피박 배수가 올바르게 적용되는지
- 멍박 배수가 올바르게 적용되는지
- 총통이 최초 분배 직후 감지되는지
- 플레이어 총통 모달과 컴퓨터 총통 모달이 맞게 표시되는지
- 흔들기 가능 시 액션 바가 표시되고 배수가 기록되는지
- 흔들기 취소 시 같은 턴에서 일반 제출로 돌아가는지
- 나가리 종료와 다음 판 배수가 저장되는지
- 쪽/따닥/뻑/싹쓸이 피 이동이 올바른지
- 폭탄 선택 팝오버, 폭탄 제출, 손패 보충이 올바른지
- 고/스톱 조건이 7점 이상에서만 열리는지
- 라운드 종료 후 상태가 초기화되는지
